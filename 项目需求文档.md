# 知讯 — 项目需求文档

> 版本：v2.0 | 更新日期：2026-06-22

---

## 一、项目概述

知讯定位为面向中小型团队的互联网内容资讯发布与管理系统，提供从内容创作、发布、审核到数据分析的全链路能力。系统采用前后端分离架构，支持多角色协作，具备智能个性化推荐、内容安全审核、用户行为分析、社交互动（关注/私信）、多端响应式适配、运营数据看板等企业级特性。

### 1.1 项目目标

- 为用户提供便捷的文章创作与发布工具
- 为运营团队提供内容审核与数据运营能力
- 为读者提供优质的内容浏览与互动体验
- 为管理层提供数据驱动的决策支持

### 1.2 用户角色

| 角色 | 编码 | 说明 |
|------|------|------|
| 超级管理员 | `SUPER_ADMIN` | 系统配置、角色权限管理、全部后台功能 |
| 运营管理员 | `ADMIN` | 内容审核、内容管理、数据查看 |
| 用户 | `USER` | 注册用户统一角色，兼具内容创作与浏览互动能力（发布文章、点赞、收藏、评论、关注、私信等） |

## 三、数据表设计

### 3.1 ER 关系概览

```
用户(1) ──< (N)文章
用户(1) ──< (N)评论
用户(1) ──< (N)浏览记录
用户(1) ──< (N)点赞
用户(1) ──< (N)收藏
用户(1) ──< (N)关注（作为关注者）
用户(1) ──< (N)粉丝（作为被关注者）
用户(1) ──< (N)私信（作为发送者）
用户(1) ──< (N)私信（作为接收者）
用户(1) ──1(1)偏好设置
分类(1) ──< (N)文章
文章(1) ──< (N)评论
文章(1) ──< (N)文章图片
标签(M) >──< (N)文章
用户(M) >──< (N)偏好分类
用户(M) >──< (N)偏好标签
```

## 四、搜索功能详细设计

### 4.1 搜索分类功能定义

#### 4.1.1 综合（默认Tab）

| 项目 | 定义 |
|------|------|
| 功能 | 聚合搜索，同时检索用户、文章、图片，按相关性加权混合排序展示 |
| 结果结构 | 顶部展示最佳匹配（1条），下方分区块展示：相关用户（≤3个）→ 相关文章（≤5篇）→ 相关图片（≤6张），每区块可"查看更多"跳转对应Tab |
| 排序算法 | 综合相关性分 = 文章相关性分 × 0.5 + 用户匹配度 × 0.3 + 图片关联度 × 0.2 |
| 无结果 | 展示热门搜索词 + 热门文章推荐 |

#### 4.1.2 用户

| 项目 | 定义 |
|------|------|
| 功能 | 按用户名、昵称、简介关键词搜索用户 |
| 结果结构 | 用户卡片：头像 + 昵称 + 简介（关键词高亮）+ 关注关系按钮 + 粉丝数/文章数 |
| 排序算法 | 精确匹配 > 前缀匹配 > 模糊匹配；同级别按粉丝数倒序；已关注用户优先展示 |
| 筛选 | 支持按粉丝数范围筛选；支持仅看已关注用户 |

#### 4.1.3 文章

| 项目 | 定义 |
|------|------|
| 功能 | 按标题、正文、标签全文检索，支持多条件组合筛选 |
| 结果结构 | 文章卡片：标题（关键词高亮）+ 摘要（关键词上下文片段高亮）+ 封面图缩略图 + 作者 + 发布时间 + 互动数据 |
| 排序算法 | 默认按相关性排序；可切换按最新发布/最多浏览/最多点赞排序 |
| 筛选 | 分类筛选（单选）、时间范围筛选（24h/7天/30天/自定义）、标签筛选 |

#### 4.1.4 图片

| 项目 | 定义 |
|------|------|
| 功能 | 搜索文章封面图及内嵌图片（基于 `cms_article_image` 表），按图片关联文章的标题/标签匹配 |
| 结果结构 | 瀑布流/网格布局：图片缩略图 + 来源文章标题 + 作者；点击图片跳转原文章 |
| 排序算法 | 按关联文章热度分倒序；同热度按图片分辨率优先 |
| 筛选 | 按分类筛选；按图片类型筛选（封面图/内嵌图） |

### 4.4 搜索算法设计

#### 4.4.1 索引架构

| 数据源 | 索引字段 | 分词策略 | 权重 |
|--------|----------|----------|------|
| 文章标题 | title | IK 中文分词（max_word） | 10 |
| 文章正文 | content | IK 中文分词（smart_word） | 3 |
| 文章标签 | tags.keyword | 精确匹配 | 8 |
| 文章分类 | category.name | 精确匹配 | 5 |
| 文章摘要 | summary | IK 中文分词（smart_word） | 6 |
| 用户名 | username | 标准分词 + 拼音分词 | 8 |
| 用户昵称 | nickname | IK 中文分词 + 拼音分词 | 10 |
| 用户简介 | bio | IK 中文分词（smart_word） | 3 |
| 图片关联 | article.title + article.tags | 继承文章索引 | — |

#### 4.4.2 相关性评分算法

**文章搜索评分公式：**

```
score = TF-IDF_score × field_weight
      + hot_score_boost × log(1 + hot_score / 100)
      + time_decay × 1 / (1 + hours_since_publish / 168)
      + personal_boost × (category_in_user_preferences ? 1.5 : 1.0)
```

| 因子 | 说明 |
|------|------|
| TF-IDF_score | OpenSearch 原生 BM25 评分 |
| field_weight | 标题匹配 ×10，标签 ×8，摘要 ×6，正文 ×3 |
| hot_score_boost | 热度加权，对数衰减避免热门文章垄断 |
| time_decay | 时间衰减因子，168h（7天）半衰期 |
| personal_boost | 用户偏好加权，感兴趣分类的文章提升 50% |

**用户搜索评分公式：**

```
score = name_match_score × field_weight
      + follower_boost × log(1 + follower_count / 100)
      + follow_priority × (is_followed_by_me ? 2.0 : 1.0)
```

| 因子 | 说明 |
|------|------|
| name_match_score | 精确匹配=100，前缀匹配=80，包含匹配=50，拼音匹配=30 |
| follower_boost | 粉丝数加权，对数衰减 |
| follow_priority | 已关注用户优先展示，2 倍加权 |

**综合搜索评分：**

```
composite_score = article_top_score × 0.5
                + user_top_score × 0.3
                + image_top_score × 0.2
```

#### 4.4.3 搜索建议算法

| 建议类型 | 数据源 | 算法 |
|----------|--------|------|
| 补全建议 | 用户搜索前缀匹配 OpenSearch Suggest 字段 | 前缀匹配 + 搜索热度排序 |
| 热门搜索 | 近 24h 搜索频次 Top 10（Redis Sorted Set） | 搜索频次统计，每小时更新 |
| 搜索历史 | 浏览器 localStorage（最近 20 条） | 按搜索时间倒序 |

### 4.5 算法性能指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| **响应时间 P95** | 综合 ≤ 300ms，文章 ≤ 200ms，用户 ≤ 150ms，图片 ≤ 250ms | OpenSearch 慢查询日志 + APM 监控 |
| **响应时间 P99** | ≤ 500ms（所有分类） | 压测工具模拟 |
| **准确率（Precision@10）** | ≥ 85%（前 10 条结果中相关结果占比） | 人工标注 500 条查询评估 |
| **召回率（Recall@100）** | ≥ 90%（前 100 条结果覆盖相关结果比例） | 人工标注 500 条查询评估 |
| **NDCG@10** | ≥ 0.80（归一化折损累积增益） | 排序质量评估 |
| **搜索建议延迟** | ≤ 100ms（输入到建议展示） | 前端性能监控 |
| **索引延迟** | ≤ 5s（文章发布到可被搜索） | 文章发布后定时验证 |
| **零结果率** | ≤ 5%（有效查询无结果的比例） | 搜索日志统计 |

### 4.6 数据来源与处理逻辑

#### 4.6.1 数据同步架构

```
MySQL (业务数据)
    │
    ├─ Canal 监听 Binlog 变更
    │     │
    │     ▼
    │   RabbitMQ (变更事件)
    │     │
    │     ▼
    │   数据清洗/转换 Worker
    │     │
    │     ▼
    │   OpenSearch (搜索索引) ← 实时增量同步，延迟 ≤ 5s
    │
    └─ 定时全量同步（每日凌晨 3:00）← 修复增量遗漏
```

#### 4.6.2 数据处理规则

| 处理环节 | 规则 |
|----------|------|
| 分词 | 中文使用 IK Analyzer（smart_word 模式）；英文使用 Standard Tokenizer；用户名额外添加拼音分词 |
| 停用词 | 移除"的/了/是/在/和"等高频无意义词 |
| 同义词 | 配置同义词表（如"AI"="人工智能"、"前端"="FE"），扩展召回 |
| HTML 过滤 | 文章正文去除 HTML 标签后索引纯文本 |
| 图片索引 | 基于 `cms_article_image` 表，关联文章标题/标签/分类作为搜索文本 |
| 热度更新 | 每小时批量更新文章 hot_score 到 OpenSearch 索引 |
| 软删除过滤 | 已删除/已下架文章不进入索引；状态变更时实时同步 |

## 六、非功能性需求

### 6.1 安全

#### 6.1.1 认证与授权

| 项目 | 要求 |
|------|------|
| 认证方式 | JWT Token 双 Token 机制：Access Token 24h + Refresh Token 7d |
| Token 存储 | Access Token 存 httpOnly Cookie（防 XSS 窃取），Refresh Token 存 httpOnly + Secure + SameSite=Strict Cookie |
| Token 刷新 | Access Token 过期前 5 分钟自动刷新；Refresh Token 轮转机制，每次刷新后旧 Token 立即失效 |
| 密码存储 | bcrypt 哈希，salt rounds ≥ 10；禁止明文存储、禁止 MD5/SHA1 |
| 密码策略 | 最少 8 位，必须包含大小写字母 + 数字；连续 5 次登录失败锁定账号 30 分钟 |
| 会话管理 | 单设备登录限制（可选）；用户主动登出时服务端将 Token 加入黑名单（Redis TTL = Token 剩余有效期） |

#### 6.1.2 越权访问防护

| 防护类型 | 实施方案 |
|----------|----------|
| **水平越权**（同角色用户间） | 所有数据操作必须校验 `当前用户ID == 资源拥有者ID`，禁止仅凭资源ID操作；文章编辑/删除需验证 `author_id == 当前用户ID` |
| **垂直越权**（低角色访问高角色功能） | 后端接口层 RBAC 权限校验，每个接口声明所需角色；管理接口统一走 `/admin` 路径前缀 + 中间件鉴权 |
| **IDOR 防护**（不安全直接对象引用） | 禁止通过遍历 ID 获取他人资源；列表接口仅返回当前用户有权查看的数据 |
| **参数篡改防护** | 关键参数（如角色、状态）不可通过请求体修改；用户角色变更仅限管理员后台操作 |

#### 6.1.3 输入验证与输出编码

| 项目 | 要求 |
|------|------|
| 服务端校验 | 所有入参必须服务端校验（类型、长度、格式、范围），不信任前端校验 |
| XSS 防护 | 富文本内容服务端 HTML 白名单过滤（允许 p/b/i/em/a/img 等安全标签）；API 响应 JSON 自动转义 `<>&'"`；CSP 策略 `default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'` |
| SQL 注入 | 全部使用参数化查询 / ORM 预编译，禁止字符串拼接 SQL |
| CSRF 防护 | SameSite Cookie + 双重提交 Cookie（请求头携带 CSRF Token）；状态变更操作（POST/PUT/DELETE）必须校验 CSRF Token |
| 文件上传 | 白名单扩展名校验（jpg/png/gif/webp/pdf）；文件头魔数校验；文件大小限制（图片 5MB，附件 20MB）；上传文件重命名，禁止保留原始文件名；存储路径不可被直接访问（走后端代理下载） |
| SSRF 防护 | 禁止用户输入直接作为服务端请求 URL；URL 白名单域名限制；禁止请求内网 IP（10.x / 172.16-31.x / 192.168.x / 127.x） |

#### 6.1.4 敏感数据加密

| 数据类型 | 加密方案 |
|----------|----------|
| 密码 | bcrypt 单向哈希（见 6.1.1） |
| 手机号/邮箱 | 数据库存储 AES-256-GCM 加密，展示时脱敏（138****1234）；密钥通过环境变量注入，禁止硬编码 |
| 身份证号 | AES-256-GCM 加密存储，任何接口不返回明文 |
| JWT Secret | 256 位随机密钥，通过环境变量注入，定期轮换 |
| 数据库连接密码 | 配置中心加密存储（如 Jasypt / Vault） |
| 传输层 | 全站 HTTPS（TLS 1.2+），HSTS 头 `max-age=31536000; includeSubDomains` |
| 私信内容 | AES-256-GCM 加密存储，仅通信双方可解密查看 |

### 6.5 高并发处理

#### 6.5.1 负载均衡

| 项目 | 方案 |
|------|------|
| 负载均衡器 | Nginx 作为七层负载均衡，支持加权轮询、IP Hash、最少连接数等策略 |
| 多实例部署 | 应用服务至少 2 个实例，通过 Docker Compose 编排；Nginx upstream 配置健康检查（`max_fails=3 fail_timeout=30s`） |
| 无状态设计 | 应用服务无状态化，Session 存 Redis，文件存 MinIO，保证任意实例可处理任意请求 |
| 会话保持 | 使用 Redis 集中管理会话，不依赖 IP Hash 粘性会话；保证实例扩缩容时会话不丢失 |
| 灰度发布 | Nginx 按 Header/Cookie 路由流量到新版本实例；支持按比例分流（如 10% 流量到新版本） |
| 优雅上下线 | 应用实例收到 SIGTERM 后停止接收新请求，等待已有请求处理完毕再关闭（Spring Boot graceful shutdown） |

#### 6.5.2 异步处理机制

| 场景 | 异步方案 |
|------|----------|
| 文章发布 | 发布请求立即返回 → MQ 异步执行敏感词检测 → 检测通过自动流转审核状态 → 审核结果通过 WebSocket/通知推送 |
| 通知推送 | 点赞/评论/关注触发通知 → MQ 异步写入 `sys_notification` 表 → WebSocket 实时推送 |
| 热度分计算 | 浏览/点赞/评论事件 → MQ 异步更新 `hot_score` → Redis Sorted Set 更新排行榜 |
| 数据统计 | 日活/浏览量/互动量 → MQ 异步聚合 → 定时任务每小时写入统计表 |
| 邮件/短信 | 验证码/通知邮件 → MQ 异步发送，失败自动重试（最多 3 次，指数退避） |
| 图片处理 | 封面图上传 → MQ 异步生成多分辨率缩略图 → 更新 `cms_article_image` 表 |

#### 6.5.3 数据库高并发

| 项目 | 方案 |
|------|------|
| 连接池 | HikariCP 连接池：最小空闲连接 10，最大连接数 50，连接超时 30s，空闲超时 10min |
| 分库分表 | 浏览记录按月分表（单表 < 500 万行）；私信按用户 ID 取模分 8 表；文章表暂不分表（预估 < 1000 万行） |

#### 6.5.4 限流降级熔断

| 项目 | 方案 |
|------|------|
| 接口限流 | 令牌桶算法（Redis + Lua），按用户 + 接口维度限流；超限返回 429 Too Many Requests |
| 熔断 | Sentinel / Resilience4j 熔断器：错误率 > 50% 或慢调用率 > 80% 时熔断 10s，半开状态探测恢复 |
| 降级策略 | 推荐服务不可用时降级为热门文章列表；搜索服务不可用时降级为数据库 LIKE 查询；排行榜不可用时返回缓存数据 |

### 6.6 安全测试

#### 6.6.1 越权访问测试

| 测试项 | 测试方法 | 预期结果 |
|--------|----------|----------|
| 水平越权 — 文章编辑 | 用户 A 尝试 PUT `/articles/{B的文章ID}` | 返回 403，无法编辑他人文章 |
| 水平越权 — 文章删除 | 用户 A 尝试 DELETE `/articles/{B的文章ID}` | 返回 403，无法删除他人文章 |
| 水平越权 — 私信读取 | 用户 A 尝试 GET `/messages/{C与D的会话}` | 返回 403，仅能查看自己的会话 |
| 垂直越权 — 管理接口 | 普通用户尝试 PUT `/admin/articles/{id}/audit` | 返回 403，无管理员权限 |
| 垂直越权 — 用户封禁 | 普通用户尝试 PUT `/admin/users/{id}/status` | 返回 403 |
| IDOR — 文章遍历 | 遍历文章 ID 1-1000 尝试访问草稿/下架文章 | 仅返回已发布且有权查看的文章 |
| 参数篡改 — 角色提升 | 用户修改请求体 `{"role": "SUPER_ADMIN"}` 尝试提权 | 角色字段被忽略，无法通过 API 提权 |

#### 6.6.2 登录校验安全测试

| 测试项 | 测试方法 | 预期结果 |
|--------|----------|----------|
| 暴力破解 | 同一 IP 连续 10 次错误密码登录 | 第 6 次起返回 429，账号锁定 30 分钟 |
| Token 窃取 | 通过 XSS 获取 Access Token | Token 存 httpOnly Cookie，JS 无法读取 |
| Token 伪造 | 修改 JWT Payload 中的用户 ID | 签名校验失败，返回 401 |
| Token 过期 | 使用过期 Access Token 请求 | 返回 401，前端自动用 Refresh Token 刷新 |
| Refresh Token 重放 | 截获 Refresh Token 后重复使用 | 轮转机制使旧 Token 立即失效 |
| 登出验证 | 登出后使用旧 Access Token 请求 | Token 已入黑名单，返回 401 |
| 并发登录 | 同一账号在两台设备同时登录 | 后登录设备生效，先登录设备 Token 失效（可选策略） |

#### 6.6.3 输入验证测试

| 测试项 | 测试方法 | 预期结果 |
|--------|----------|----------|
| XSS — 存储型 | 文章内容插入 `<script>alert(1)</script>` | 脚本被过滤，不执行；内容安全渲染 |
| XSS — 反射型 | URL 参数注入 `<script>` 标签 | 参数被转义，不执行 |
| XSS — DOM 型 | 前端动态渲染用户输入 | 使用 `v-text` 而非 `v-html`，自动转义 |
| SQL 注入 | 搜索框输入 `' OR 1=1 --`、`'; DROP TABLE --` | 参数化查询，注入无效，返回正常结果或空 |
| CSRF 攻击 | 伪造跨站 POST 请求 | 无 CSRF Token 请求被拒绝（403） |
| 文件上传 — 恶意文件 | 上传 .php/.jsp 文件伪装为图片 | 扩展名校验 + 文件头校验，拒绝上传 |
| 文件上传 — 大文件 | 上传 100MB 文件 | 超过大小限制，返回 413 |
| 路径遍历 | 请求 `../../../etc/passwd` | 路径规范化处理，返回 404 |
| 命令注入 | 输入 `; rm -rf /` | 输入被转义，不执行系统命令 |

#### 6.6.4 敏感数据安全测试

| 测试项 | 测试方法 | 预期结果 |
|--------|----------|----------|
| 密码明文 | 查询数据库 `sys_user.password_hash` | 值为 bcrypt 哈希，非明文 |
| 手机号脱敏 | 调用用户信息接口 | 返回 `138****1234` 格式，非明文 |
| 私信加密 | 直接查询数据库 `user_message.content` | 内容为 AES 密文，非明文 |
| HTTPS 强制 | HTTP 请求访问任意接口 | 301 重定向到 HTTPS |
| HSTS | 首次 HTTPS 访问后，关闭 HTTPS 再访问 | 浏览器强制 HTTPS，拒绝 HTTP 连接 |
| 密钥泄露 | 搜索代码仓库中的硬编码密钥 | 无硬编码密钥，全部通过环境变量注入 |

#### 6.6.5 权限控制测试

| 测试项 | 测试方法 | 预期结果 |
|--------|----------|----------|
| 未认证访问 | 不携带 Token 请求需认证接口 | 返回 401 |
